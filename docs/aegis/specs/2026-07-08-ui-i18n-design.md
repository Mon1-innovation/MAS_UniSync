# UI i18n and Submod Chinese Copy Design

Date: 2026-07-08

## Goal

Translate MAS UniSync's user-facing UI to Chinese by default while preserving an
English option in the web portal. Translate the MAS/Ren'Py submod UI to fixed
Chinese.

## Scope

In scope:

- Web portal navigation, page headings, buttons, tables, dialogs, empty states,
  validation messages, error messages, loading text, and accessible labels.
- Web portal language switch between Chinese and English.
- Persisting the selected web portal language locally in the browser.
- Submod setting panel labels, buttons, notifications, dialogs, visible status
  labels, and user-facing error prefixes.
- Tests for default Chinese behavior and English switching.

Out of scope:

- Translating API paths, database fields, JSON keys, profile keys, audit action
  codes, or protocol constants.
- Adding language switching to the submod.
- Changing backend behavior or storage contracts.
- Translating developer documentation beyond this design record.

## Design

The web portal will use `i18next` and `react-i18next`.

- Default language is Chinese (`zh`).
- English (`en`) remains available through a compact language switch in the app
  header.
- The selected language is saved in `localStorage`.
- Translation resources live under `frontend/src/i18n/`.
- React components read strings with `useTranslation()`.
- Machine values remain unchanged and are mapped at display boundaries. Examples
  include role labels, profile status, and API error codes.

The submod will be fixed Chinese.

- User-facing strings in `game/Submods/MAS_UniSync/header.rpy` are translated to
  Chinese.
- Internal status values such as `enabled`, `disabled`, `locked`, and `unlocked`
  stay unchanged for compatibility.
- The settings pane maps those internal values to Chinese display labels.
- API key feature names, URL values, profile keys, and server protocol behavior
  are not changed.

## Compatibility

- Frontend routing and API contracts remain unchanged.
- Existing browser cached users remain valid.
- Language preference is additive browser state and can be reset by clearing
  local storage.
- Submod changes must remain compatible with current Ren'Py/Python syntax.

## Testing

Frontend tests should cover:

- Chinese is the default language.
- The language switch changes visible UI to English.
- The selected language persists after rerender or reload.
- Existing workflows still submit the same request payloads.

Verification commands:

```powershell
cd frontend
npm test
npm run typecheck
```

Submod verification should include existing Python tests that exercise submod
packaging and client behavior:

```powershell
pytest tests/test_submod_packaging.py tests/test_submod_client_core.py
```
